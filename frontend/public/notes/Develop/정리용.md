- 동시성 = 싱글코어, 컨텍스트 스위칭, I/O 대기 등 유휴시간 활용

Branch : Git 저장소에서 개발 작업을 분리하기 위한 독립된 작업 흐름

기존 코드베이스에서 새로운 작업을 시작하거나 기능을 개발할 때 Branch를 씀.

각 Branch는 고유한 commit 기록을 가지며, 서로 독립적으로 작업을 진행할 수 있다.

Commit :Git에서 코드의 변경 사항을 저장하는 작업 단위

변경된 파일들의 snapshot을 생성하고 메시지와 함께 저장소에 기록된다.

Commit 은 고유한 식별자(hash)를 가지며, 이를 통해 이전 commit들과의 관계를 추적할 수 있다.

Merge : 서로 다른 브랜치에서 개발한 내용을 하나로 통합하는 작업

다른 브랜치에서의 Commit 내용을 현재 Branch로 가져와서 합친다.

Git은 자동으로 병합을 시도하고, Merge 과정에서 충돌(conflict)이 발생할 수 있다.

Pull : 원격 저장소에서 최신 변경 사항을 가져오는 작업

Local Branch에 있는 작업과 원격 저장소의 Branch 간의 차이를 확인하고, 변경 사항을 가져온다.

pull은 fetch(원격 저장소에서 변경 사항을 가져오기)와

merge(가져온 변경 사항을 병합하기)를 일반적으로 자동으로 수행한다

Checkout: 작업 디렉토리를 특정 commit, branch 또는 태그로 변경하는 작업

다른 Branch로 전환하거나 특정 Commit의 상태를 확인하고 싶을 때 사용한다다.

작업 트리와 인덱스를 변경하여 해당 상태로 이동한다.
