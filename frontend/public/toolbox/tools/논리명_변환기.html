버튼내용
매칭결과 CSV모든 행의 스키마·테이블·컬럼·논리명·근거·미매칭토큰

상단 DB 방언 탭(Tibero/Oracle·MySQL·PostgreSQL·SQL Server·Sybase)이 ②의 조회쿼리와 코멘트 DDL 문법을 결정한다. 같은 논리명이라도 방언마다 문법이 다르다. 대상 DB 방언을 고른다.

1. 공통표준단어

항목,ITEM + 아이디,ID → ITEM_ID 를 항목아이디 로 조립
파일은 1개만 넣는다. 새로 넣으면 이전 것은 버려진다.
파일 받는 곳 — 공공데이터포털, 로그인 불필요
검색어 공공데이터 공통표준단어 또는 아래 주소
https://www.data.go.kr/data/15156439/fileData.do←행정안전부_공공데이터 공통표준단어_YYYYMMDD.csv
⬇ 파일을 여기로 끌어다 놓아도 된다
파일을 넣으면 자동 반영된다. 열 선택을 고쳤으면 2번의 [변환 실행] 을 다시 누른다.

1-2. 기관표준단어 (선택)

ITEM_ID,항목ID → 조립하지 않고 항목ID 를 그대로 채택. 최우선.
회사(기관)가 자체적으로 쓰는 표준단어·표준용어 — 공통표준단어의 기관 버전
공통표준단어(표준단어 CSV)는 여기 말고 1번에.
⬇ 파일을 여기로 끌어다 놓아도 된다
← 컬럼명 통째 매칭은 항상 1-2번이 우선. 이 설정은 토큰 조회에만 적용

2. 컬럼 목록 (컬럼 1건 = 1행)

상단 방언 탭 조회쿼리(ALL_TAB_COLUMNS 등) 결과, 또는 회사 테이블정의서·컬럼정의서를 CSV로 넣는다.
컬럼 1건 = 1행 (테이블명이 반복됨).
테이블 목록은 따로 넣지 않는다 — TABLE_NAME 을 중복 제거해 자동 도출. 자세한 형식은 0번 입력 형식 안내 참고.
⬇ 파일을 여기로 끌어다 놓아도 된다
← CSV에 열이 없으면 (직접 입력) 선택
← 조회쿼리에 TABLE_NAME 포함되니 항상 있다
쉼표 구분 · 테이블 논리명 만들 때 건너뜀

3. 변환 결과

논리명은 여기서 고치지 않는다. 표준단어로 조립된 결과다.
바꾸려면 4번 랭킹에 약어별 한글을 넣거나, 1-2번 기관표준단어에 물리명을 통째로 등록한다. 그래야 다음 변환에도 남는다.

4. 미등록 약어 랭킹 — 빈도순

한글을 입력하면 즉시 재변환한다. 입력값은 이 브라우저에 저장되어 다음에 열어도 남는다. 한 번 넣은 약어는 매칭에 성공해 미매칭이 아니게 되므로 출현 0 으로 목록에 남는다 — 사라진 게 아니다.
[랭킹 CSV 저장] 은 영문약어,한글입력 순서로 나온다 — 1-2번 기관표준단어에 그대로 넣거나 기존 정의서 뒤에 붙여도 된다.

5. 출력

스키마명 항상 포함 (OWNER.TABLE)
생성물은 텍스트다. 운영 DB 실행은 별도 승인 사항.
컬럼명만 모드 — 테이블명 열이 (없음).
컬럼 코멘트는 테이블명이 필요해(방언 무관) DDL 생성을 막았다. 매칭결과 CSV는 저장된다.
← 용어사전=테이블+컬럼 distinct(구분열) · 단어사전=3소스 통합(공통Y+기관+랭킹)
